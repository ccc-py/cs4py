# 忙碌海狸問題 (Busy Beaver Problem)

## 歷史背景

忙碌海狸問題 (Busy Beaver Problem) 由 Tibor Radó 在 1962 年提出，是一個著名的不可判定問題。

### 重要里程碑

- **1962 年**：Tibor Radó 提出忙碌海狸問題
- **Σ(n) 的成長**：超越任何可計算函數
- **目前已知**：只證明到 n=4，n≥5 的值未知

## 核心概念

### 問題描述

> 給定一個有 n 個狀態的圖靈機（加上起始、接受、拒絕狀態），在空白帶（全為 0）上開始執行，最多可以寫出多少個 1 才停機？

### 定義

Σ(n) = 有 n 個狀態的圖靈機在空白帶上能寫出的最大 1 的數量

### 已知值

| n | Σ(n) | 步數 | 發現者 |
|---|-------|------|---------|
| 1 | 1 | 1 | Radó (1962) |
| 2 | 4 | 6 | Radó (1962) |
| 3 | 6 | 14 | Lin & Radó (1965) |
| 4 | 13 | 107 | Brady (1975) |
| 5 | ≥ 4098 | ≥ 47,176,870 | Marxen & Buntrock (1989) |
| 6 | ≥ 3.5 × 10¹⁸²⁶⁷ | 未知 | Kropitz (2010) |

### Σ(n) 的特性

**超越任何可計算函數**：
- 不存在任何可計算函數 f(n) 使得 Σ(n) ≤ f(n) 對所有 n 成立
- 如果 Σ(n) 是可計算的，那麼停機問題就可判定 → 矛盾！

## 程式碼說明

### 2-狀態忙碌海狸

`create_busy_beaver_2()` 實作了 Σ(2)=4 的圖靈機：

**轉移規則**：
```
狀態\符號 | 0 | 1
-----------|---|---
A          | 1R B | 1L HALT
B          | 1L A | 1R HALT
```

**執行過程**：
```
帶子: ...000000...
步驟 1: A 讀 0 → 寫 1, 右移, 去 B
步驟 2: B 讀 0 → 寫 1, 左移, 去 A
步驟 3: A 讀 0 → 寫 1, 右移, 去 B
步驟 4: B 讀 0 → 寫 1, 右移, 去 HALT
```

結果：寫出 4 個 1 後停機。

### 3-狀態忙碌海狸

`create_busy_beaver_3()` 實作了 Σ(3)=6 的圖靈機。

## 忙碌海狸與不可判定性

如果 Σ(n) 是可計算的，那麼：
1. 對於任意 n-狀態圖靈機，我們可以計算 Σ(n)
2. 執行該圖靈機 Σ(n) 步
3. 如果還沒停機，則它永遠不會停機（因為最多寫 Σ(n) 個 1）
4. 這樣我們就解決了停機問題 → **矛盾！**

因此，**Σ(n) 不可計算**。
