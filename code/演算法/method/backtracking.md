# 回溯法 (Backtracking)

## 回溯法範式

回溯法是一種系統化的搜尋方法，透過嘗試所有可能的候選解來求解問題。當發現當前路徑不可能導向有效解時，就「回溯」到上一步，嘗試其他選擇。

### 核心思想

> 「試錯法」的系統化實現

- **探索**：深度優先搜尋（DFS）遍歷解空間樹
- **驗證**：在每個節點檢查部分解是否有效
- **剪枝 (Pruning)**：及早發現無效路徑，避免無謂搜尋

## 通用框架

```python
def backtrack(解空間, 部分解):
    if 滿足結束條件:
        記錄解
        return
    
    for 每個可能的選擇:
        if 選擇有效:
            做出選擇
            backtrack(解空間, 部分解)
            撤銷選擇  # 回溯
```

## 經典問題

### 1. N-皇后問題 (N-Queens)

**問題**：在 n×n 的棋盤上放置 n 個皇后，使得它們互不攻擊（不在同一行、列或對角線）。

**解法思路**：
1. 逐行放置皇后
2. 每行嘗試所有可能的列
3. 用檢查函數確保位置安全
4. 找到 n 個皇后後記錄解

**時間複雜度**：最壞 O(n!)，但實際透過剪枝會快很多

**優化技巧**：使用位元運算表示列和對角線的佔用情況

### 2. 數獨求解器 (Sudoku Solver)

**問題**：填滿 9×9 的數獨棋盤，滿足每行、每列、每個 3×3 宮格都包含 1-9。

**解法思路**：
1. 找到空格
2. 嘗試填入 1-9
3. 檢查是否符合數獨規則
4. 遞迴求解剩餘空格

**時間複雜度**：取決於空格數量，最壞情況下是指數級

## 剪枝策略

有效的剪枝可以大幅減少搜尋空間：

| 問題 | 剪枝條件 |
|------|---------|
| N-皇后 | 列、對角線已被佔用 |
| 數獨 | 行、列、宮格已有該數字 |
| 背包問題 | 剩餘容量不足 |
| 排列組合 | 已使用的元素 |

## 回溯法 vs 其他搜尋方法

| 方法 | 搜尋策略 | 適用場景 |
|------|---------|---------|
| 回溯法 | DFS + 剪枝 | 約束滿足問題 |
| 分支限界法 | BFS/優先佇列 + 剪枝 | 最佳化問題 |
| 動態規劃 | 自底向上 | 重疊子問題 |

## 應用場景

1. **約束滿足問題 (CSP)**：數獨、地圖著色
2. **組合最佳化**：旅行推銷員（精確解）
3. **棋盤遊戲**：國際象棋、圍棋（結合α-β剪枝）
4. **路徑規劃**：迷宮求解
5. **排列組合生成**：子集、排列、組合

## 常見優化

1. **啟發式排序**：優先嘗試更有希望的選擇
2. **前向檢查 (Forward Checking)**：預先看未來是否可行
3. **約束傳播 (Constraint Propagation)**：動態更新可行域
4. **記憶化**：記錄已探索的狀態
