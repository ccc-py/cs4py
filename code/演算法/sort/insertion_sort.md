# 插入排序與變體 (Insertion Sort and Variants)

## 歷史背景

插入排序是最自然、最直觀的排序演算法之一，類似於人類整理撲克牌的方式。

### 發展歷程

- **古代**：人類早已使用類似方法整理物品
- **1945 年**：John von Neumann 在排序理論的開創性工作中描述了插入排序
- **1959 年**：Donald Shell 提出了 Shell 排序，是插入排序的重要改進
- **現代**：插入排序在小資料量和近乎有序的場景下仍然非常有用

## 演算法原理

### 標準插入排序

```
過程（類似整理撲克牌）：
1. 將陣列分為「已排序」和「未排序」兩部分
2. 初始時，第一個元素視為已排序
3. 從未排序部分取出第一個元素
4. 在已排序部分從後往前掃描，找到合適位置插入
5. 重複直到未排序部分為空
```

**圖示**：
```
初始：[64, 34, 25, 12, 22]
步驟1：[64 | 34, 25, 12, 22]  -> [34, 64 | 25, 12, 22]
步驟2：[34, 64 | 25, 12, 22]  -> [25, 34, 64 | 12, 22]
步驟3：[25, 34, 64 | 12, 22]  -> [12, 25, 34, 64 | 22]
步驟4：[12, 25, 34, 64 | 22]  -> [12, 22, 25, 34, 64]
```

### 二分插入排序

使用二分搜尋來減少比較次數，但移動元素的次數不變。

```
改進：
- 標準插入排序：比較次數 O(n²)
- 二分插入排序：比較次數 O(n log n)
- 但移動次數仍是 O(n²)，所以整體仍是 O(n²)
```

### Shell 排序

Shell 排序是插入排序的推廣，通過允許交換距離較遠的元素來改進效率。

```
核心概念：
1. 使用間隔序列（gap sequence）
2. 先用大間隔進行部分排序，讓元素快速接近正確位置
3. 逐漸縮小間隔，最後間隔為 1 時就是插入排序

間隔序列（Knuth）：
  gap = 1, 4, 13, 40, 121, ...
  公式：gap = gap * 3 + 1
  縮小：gap = gap // 3
```

## 複雜度分析

| 演算法 | 最佳 | 平均 | 最壞 | 空間 | 穩定 |
|--------|------|------|------|------|------|
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | 是 |
| 二分插入排序 | O(n log n) | O(n²) | O(n²) | O(1) | 是 |
| Shell 排序 | O(n log n) | O(n^1.5) | O(n²) | O(1) | 否 |

## 自適應特性

插入排序是自適應排序（Adaptive Sort），其效能取決於輸入資料的初始有序程度。

```
比較次數：
- 已排序陣列：n-1 次比較（最佳情況）
- 反序陣列：n(n-1)/2 次比較（最壞情況）
- 部分有序：介於兩者之間
```

這使得插入排序在處理近乎有序的資料時非常高效。

## 程式碼說明

### 插入排序核心

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]  # 移動元素
            j -= 1
        arr[j + 1] = key  # 插入
```

### Shell 排序核心

```python
def shell_sort(arr):
    n = len(arr)
    gap = 1
    while gap < n // 3:
        gap = gap * 3 + 1  # Knuth 序列

    while gap > 0:
        for i in range(gap, n):
            temp = arr[i]
            j = i
            while j >= gap and arr[j - gap] > temp:
                arr[j] = arr[j - gap]
                j -= gap
            arr[j] = temp
        gap //= 3
```

## 應用場景

### 1. 小資料量排序

當 n ≤ 50 時，插入排序往往比快速排序等更簡單高效。

### 2. 近乎有序的資料

如果資料已經大致有序，插入排序非常快（接近 O(n)）。

### 3. 線上演算法

資料流式到達時，插入排序可以維護已排序序列。

### 4. 作為其他演算法的子程序

許多高級排序演算法（如 Timsort）在小片段使用插入排序。

## 與其他排序比較

| 特性 | 插入排序 | 合併排序 | 快速排序 |
|------|---------|---------|---------|
| 最好情況 | O(n) | O(n log n) | O(n log n) |
| 原地排序 | 是 | 否 | 是 |
| 穩定排序 | 是 | 是 | 否 |
| 自適應 | 是 | 否 | 否 |
