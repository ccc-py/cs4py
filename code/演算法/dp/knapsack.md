# 背包問題 (Knapsack Problem)

## 歷史背景

背包問題是組合優化中的經典問題，最早可追溯到 1897 年數學家 Tobias Dantzig 的研究。問題名稱來自於「在限定容量的背包中放入最有價值的物品」這一形象比喻。

0/1 背包問題是 NP-Complete 問題，但對於較小的容量，可以使用動態規劃在多項式時間內求解。

## 核心原理

### 0/1 背包問題

每個物品只能選取一次（選或不選）。

**狀態定義**：`dp[i][w]` 表示考慮前 i 個物品，在容量為 w 時的最大價值。

**狀態轉移方程**：
```
dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])
```

**時間複雜度**：O(n × capacity)
**空間複雜度**：二維版本 O(n × capacity)，優化版本 O(capacity)

### 無界背包問題

每個物品可以選取無限次，狀態轉移時允許重複選取同一物品。

**關鍵區別**：無界背包在第二層循環時從小到大遍歷（允許重複選取），而 0/1 背包必須從大到小遍歷（避免重複選取）。

## 實作說明

### 檔案結構

- `knapsack_01()`：標準二維 DP 解法，便於理解
- `knapsack_01_optimized()`：空間優化的一維陣列解法
- `knapsack_unbounded()`：無界背包解法

### 回溯重建解

DP 表只能得到最大價值，要得到具體選取了哪些物品，需要從 `dp[n][capacity]` 回溯：
- 若 `dp[i][w] != dp[i-1][w]`，表示選取了第 i 個物品
- 減去該物品重量，繼續回溯

## 複雜度分析

| 版本 | 時間複雜度 | 空間複雜度 |
|------|-----------|-----------|
| 0/1 二維 | O(n × C) | O(n × C) |
| 0/1 優化 | O(n × C) | O(C) |
| 無界背包 | O(n × C) | O(C) |

其中 n 為物品數量，C 為背包容量。

## 應用場景

- 資源分配問題
- 投資組合優化
- 貨物裝載問題
- 預算分配
