# Okapi BM25 排名函數

## 歷史背景

BM25（Best Matching 25）由 Stephen Robertson 和 Karen Spärck Jones 在 1990 年代初期提出，是基於機率論檢索模型（Probabilistic Relevance Model）的排名函數。BM25 是 BM 系列的第 25 個版本，至今仍是搜尋引擎標準的排名演算法之一。

BM25 結合了：
- TF-IDF 的優點
- 詞頻飽和度（term frequency saturation）
- 文件長度正規化

現代搜尋引擎（如 Elasticsearch、Lucene）均使用 BM25 或其變體作為預設排名函數。

## 核心原理

### BM25 公式

```
score(D, Q) = Σ IDF(qi) * (tf * (k1 + 1)) / (tf + k1 * (1 - b + b * |D|/avgdl))
```

其中：
- `tf`：詞彙在文件中的頻率
- `k1`：詞頻飽和參數（通常 1.2-2.0），控制詞頻的影響
- `b`：長度正規化參數（通常 0.75），控制文件長度的影響
- `|D|`：文件長度
- `avgdl`：平均文件長度

### IDF 部分

BM25 使用改進的 IDF 公式：
```
IDF(qi) = log((N - df + 0.5) / (df + 0.5) + 1)
```

### 詞頻飽和度

詞頻對分數的影響隨著頻率增加而遞減，避免過度懲罰高頻詞。

### 文件長度正規化

較長的文件會被輕微懲罰，因為長文件傾向涵蓋更多主題。

## 使用範例

```python
from ranking.bm25 import BM25

docs = ["the cat sat", "the dog ran fast", "cat and dog"]
bm25 = BM25(k1=1.5, b=0.75)
bm25.fit(docs)

rankings = bm25.rank("cat dog")
for idx, score in rankings:
    print(f"文件 {idx}: {score:.4f}")
```
