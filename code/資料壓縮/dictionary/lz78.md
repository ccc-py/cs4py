# LZ78 壓縮演算法

## 歷史背景

LZ78 由 Abraham Lempel 和 Jacob Ziv 在 1978 年提出，是繼 LZ77 之後的另一個重要的字典式壓縮演算法。LZ78 與 LZ77 的主要區別在於：

- **LZ77** 使用滑動視窗（隱式字典）
- **LZ78** 使用顯式字典（通常實作為前綴樹/Trie）

LZ78 的變體 LZW（加上 Terry Welch 的改進）後來成為 GIF 格式的壓縮演算法。LZ78 本身也被用於各種應用中。

## 核心原理

### 字典結構

LZ78 維護一個顯式字典，將字串對應到索引。字典隨著壓縮過程動態增長。

### 壓縮過程

以 "TOBEORNOT" 為例：

1. 初始字典為空
2. 讀取 'T'，不在字典中，輸出 `(0, 'T')`，將 "T" 加入字典（索引 1）
3. 讀取 'O'，不在字典中，輸出 `(0, 'O')`，將 "O" 加入字典（索引 2）
4. 讀取 'B'，不在字典中，輸出 `(0, 'B')`，將 "B" 加入字典（索引 3）
5. 讀取 'E'，不在字典中，輸出 `(0, 'E')`，將 "E" 加入字典（索引 4）
6. 讀取 'O'，在字典中（索引 2），繼續讀 'R'
7. "OR" 不在字典中，輸出 `(2, 'R')`，將 "OR" 加入字典（索引 5）
8. ...依此類推

### 輸出格式

每個 token 是 `(索引, 字元)` 二元組：
- **索引**：指向字典中已存在的前綴
- **字元**：新字元，與前綴組成新字串加入字典

### 解壓縮過程

解壓縮端同步建立字典：
1. 讀取 `(索引, 字元)`
2. 從字典中取出索引對應的字串 prefix
3. 輸出 `prefix + 字元`
4. 將 `prefix + 字元` 加入字典

## 使用範例

```python
from code.資料壓縮.dictionary.lz78 import encode, decode, tokens_to_string

# 原始資料
text = "TOBEORNOTTOBEORTOBEORNOT"

# 壓縮
tokens = encode(text)
print("壓縮 tokens:", tokens_to_string(tokens))
print(f"Token 數量: {len(tokens)}")
print(f"原始長度: {len(text)} 字元")

# 解壓縮
decoded = decode(tokens)
print(f"解壓後: {decoded}")
print(f"驗證正確: {decoded == text}")
```

## LZ77 vs LZ78

| 特性 | LZ77 | LZ78 |
|------|------|------|
| 字典類型 | 滑動視窗（隱式） | 前綴樹（顯式） |
| 搜尋方式 | 在視窗中搜尋匹配 | 在前綴樹中搜尋 |
| 壓縮速度 | 較慢（字串匹配） | 較快 |
| 適用場景 | 通用資料 | 文字資料 |
