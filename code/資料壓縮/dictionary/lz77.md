# LZ77 滑動視窗壓縮

## 歷史背景

LZ77 由 Abraham Lempel 和 Jacob Ziv 在 1977 年發表的論文《A Universal Algorithm for Sequential Data Compression》中提出。這是第一個實用的**字典式壓縮**（Dictionary-based Compression）演算法。

LZ77 的核心創新在於使用**滑動視窗**（Sliding Window）來動態地建立字典，而不需要預先傳送字典。這個想法非常優雅：利用已經處理過的資料作為字典，來壓縮後續的資料。

LZ77 的變體被廣泛應用於：
- **DEFLATE**（ZIP、gzip、PNG）
- **LZMA**（7-Zip）
- **LZX**（Microsoft CAB）

## 核心原理

### 滑動視窗結構

LZ77 維護一個滑動視窗，分為兩個部分：

```
┌─────────────────┬──────────────────┐
│   搜尋視窗       │    前瞻視窗      │
│  (已處理資料)    │  (待處理資料)    │
└─────────────────┴──────────────────┘
```

- **搜尋視窗**（Search Window）：存放已經處理過的資料，用來尋找匹配
- **前瞻視窗**（Lookahead Window）：存放還沒處理的資料

### 三元組表示法

LZ77 將資料壓縮為一系列三元組：

\[
(offset, length, next\_char)
\]

- **offset**：匹配位置距離當前位置的偏移量
- **length**：匹配的長度
- **next_char**：匹配結束後的下一個字元（如果沒有匹配，則 length=0，next_char 為當前字元）

### 壓縮過程

以 "abracadabra" 為例：

1. 初始時，搜尋視窗為空，輸出 `'a'`
2. 處理 `'b'`：無匹配，輸出 `'b'`
3. ...依此類推
4. 當處理到第二個 "abra" 時，可以在搜尋視窗中找到匹配，輸出 `(7, 4, ' ')` 表示從 7 個位置前開始，匹配 4 個字元，然後是空格

## 使用範例

```python
from code.資料壓縮.dictionary.lz77 import encode, decode, tokens_to_string

# 原始資料
text = "abracadabra abracadabra"

# 壓縮
tokens = encode(text, window_size=20, lookahead_size=15)
print("壓縮 tokens:", tokens_to_string(tokens))
print(f"Token 數量: {len(tokens)}")
print(f"原始長度: {len(text)} 字元")

# 解壓縮
decoded = decode(tokens)
print(f"解壓後: {decoded}")
print(f"驗證正確: {decoded == text}")
```

### 執行結果範例

```
壓縮 tokens: 'a' 'b' 'r' (3,1,'c') (4,1,'d') (7,5,' ')
原始長度: 25 字元
Token 數量: 10
解壓後: abracadabra abracadabra
驗證正確: True
```

## 優缺點

### 優點
- 不需要預先傳送字典
- 對於重複性高的資料壓縮效果好
- 演算法簡單，易於實作

### 缺點
- 搜尋匹配較慢（可用雜湊表優化）
- 對於隨機資料壓縮效果有限
