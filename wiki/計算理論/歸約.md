# 多項式時間歸約 (Polynomial-Time Reduction)

## 歷史背景

歸約是計算複雜度理論中的核心概念，由 Stephen Cook 在 1971 年用於證明 SAT 是 NP-完備的。

### 重要里程碑

- **1971 年**：Cook 使用歸約證明 SAT 是 NP-完備的
- **1972 年**：Karp 使用歸約證明了 21 個問題是 NP-完備的
- **至今**：歸約建立了數千個 NP-完備問題之間的聯繫網絡

## 核心概念

### 定義

A ≤p B（A 多項式時間歸約到 B）表示：
- 存在多項式時間可計算函數 f
- 使得：x ∈ A ⇔ f(x) ∈ B

### 意義

```
如果 B ∈ P，則 A ∈ P
（因為可以用 f 將 A 轉為 B，然後用 B 的多項式演算法）
```

### 重要性

歸約是證明 NP-hard 的標準方法：
1. 已知某個 NP-完備問題 A
2. 證明 A ≤p B
3. 因此 B 是 NP-hard
4. 如果 B ∈ NP，則 B 是 NP-完備的

## 歸約範例

### 3-SAT ≤p Clique

**思路**：
1. 每個子句對應一個組（組的大小 = 子句長度）
2. 每個文字是一個頂點：(子句索引, 文字)
3. 連接「相容」的頂點對（不同子句，且文字不矛盾）
4. 存在大小 = 子句數的團 ⇔ 公式可滿足

### Clique ≤p Vertex Cover

**定理**：G 有大小為 k 的團 ⇔ 補圖 G' 有大小為 |V|-k 的頂點覆蓋。

**思路**：
1. 團中的頂點在補圖中沒有邊相連
2. 因此這些頂點在補圖中需要被覆蓋的邊都不與它們相連
3. 只需要覆蓋剩下的頂點

## 歸約的性質

1. **遞移性**：如果 A ≤p B 且 B ≤p C，則 A ≤p C
2. **如果 B ∈ P 且 A ≤p B，則 A ∈ P**
3. **如果 A 是 NP-hard 且 A ≤p B，則 B 也是 NP-hard**

## NP-完備性證明概要

要證明問題 L 是 NP-完備的：

1. **證明 L ∈ NP**：
   - 給定一個「證明」（解），存在多項式時間驗證器
   - 驗證器檢查該證明是否正確

2. **證明 L 是 NP-hard**：
   - 選擇一個已知的 NP-完備問題 A（如 SAT）
   - 構造多項式時間歸約 f: A → L
   - 證明：x ∈ A ⇔ f(x) ∈ L

## 歸約鏈

```
SAT ≤p 3-SAT ≤p Clique ≤p Vertex Cover ≤p Hamiltonian Cycle ≤p TSP
```

如果 TSP ∈ P，則所有上述問題 ∈ P！
